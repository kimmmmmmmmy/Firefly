# 博客更新流程

> 每次更新博客，按以下三步操作即可。

---

## 流程概览

```
本地修改 → 推送到 GitHub → 服务器拉取并构建
```

---

## 第一步：本地修改博客

在本地编辑博客内容，例如：

- 在 `src/content/posts/` 目录下新增或修改文章（`.md` 文件）
- 修改配置文件 `astro.config.mjs`
- 修改样式或页面代码

---

## 第二步：推送到 GitHub

在本地项目目录 `F:\blog\Firefly-master\Firefly-master` 打开终端，运行：

```powershell
# 1. 进入项目目录（注意要先切换到 F 盘）
F:
cd "F:\blog\Firefly-master\Firefly-master"

# 2. 添加所有修改
git add .

# 3. 提交，引号里写本次修改的说明
git commit -m "新增文章：文章标题"

# 4. 推送到 GitHub
git push origin main
```

推送成功后，可以去 [https://github.com/kimmmmmmmmy/Firefly](https://github.com/kimmmmmmmmy/Firefly) 确认代码已更新。

---

## 第三步：服务器拉取并重新构建

登录服务器（宝塔面板 → 终端），运行：

```bash
# 1. 进入项目目录
cd /www/wwwroot/blog

# 2. 从 GitHub 拉取最新代码
git pull origin main

# 3. 重新构建，生成最新静态文件
pnpm run build
```

构建完成后，网站即自动更新，访问 [https://blog.aixiaobo.cn](https://blog.aixiaobo.cn) 查看效果。

---

## 常用 Git 命令速查

| 命令 | 说明 |
|------|------|
| `git status` | 查看哪些文件被修改了 |
| `git add .` | 添加所有修改的文件 |
| `git commit -m "说明"` | 提交并写备注 |
| `git push origin main:master` | 推送到 GitHub（本地 main → 远程 master） |
| `git pull origin master` | 从 GitHub 拉取最新代码（服务器执行） |

---

## 后续优化（可选）

可以配置 **GitHub Actions 自动部署**，代码 push 后服务器自动拉取并构建，无需手动登录服务器执行第三步。
